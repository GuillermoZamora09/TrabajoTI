Puertos de red

Qué son los puertos de red TCP y UDP

Los puertos de red TCP (Protocolo de Control de Transmisión) y UDP (Protocolo de Datagramas de Usuario) son números de identificación que se utilizan en las comunicaciones de red para permitir que las aplicaciones y servicios se comuniquen entre sí de manera efectiva en una red de computadoras. Cada dispositivo en una red tiene una dirección IP única que le permite ser identificado en la red, y los puertos TCP y UDP permiten a los dispositivos distinguir qué aplicación o servicio específico en ese dispositivo debe manejar los datos entrantes.

TCP Cuando nos queremos conectar a Internet, solo necesitamos un par de maniobras para establecer una conexión entre el rúter y el ordenador o el terminal móvil. No se requieren más pasos, ya que el inicio de sesión en red es automático, al igual que la asignación de una dirección individual de Internet que necesitas para enviar y recibir datos. Todo ello es posible gracias a una serie de protocolos que también se conoce como familia de protocolos de Internet. En este contexto, uno de los miembros más importantes y antiguos es el protocolo de control de transmisión TCP, siglas de Transmission Control Protocol, y sirve para determinar cómo los dispositivos reunidos en la red deben transmitir sus datos.

  • HTTP (Hypertext Transfer Protocol) Puerto TCP 80
  • HTTPS (Hypertext Transfer Protocol Secure) Puerto TCP 443
  • SMTP (Simple Mail Transfer Protocol) Puerto TCP 25
  • POP3 (Post Office Protocol version 3) Puerto TCP 110
  • IMAP (Internet Message Access Protocol) Puerto TCP 143
  • MIME (Multipurpose Internet Mail Extensions) MIME en sí mismo no utiliza un puerto específico, ya que es una especificación para el formato de los mensajes de correo electrónico. Sin embargo, se utiliza junto con SMTP, POP3, o IMAP en sus respectivos puertos.
    • FTP (File Transfer Protocol) Puerto TCP 21 para el control Puerto TCP 20 para la transferencia de datos (modo activo)
  • FTP (File Transfer Protocol) Puerto TCP 21 para el control Puerto TCP 20 para la transferencia de datos (modo activo)
  • FTPS (FTP Secure) Puertos TCP 989 (control) y 990 (datos) para FTPS explícito Puertos TCP 990 (control) y 989 (datos) para FTPS implícito
  • SFTP (SSH File Transfer Protocol) SFTP generalmente se ejecuta sobre SSH (Puerto TCP 22). No tiene un puerto dedicado.
  • BitTorrent BitTorrent utiliza una variedad de puertos TCP y UDP, generalmente en el rango de 6881-6889, pero puede variar.
  • eDonkey Puerto TCP 4662
  • FastTrack (utilizado por Kazaa) Puerto TCP 1214
  • IRC (Internet Relay Chat) Puerto TCP 6660-6669 (no seguro) Puerto TCP 6697 (seguro)
  • XMPP (Extensible Messaging and Presence Protocol) Puerto TCP 5222 (no seguro) Puerto TCP 5223 (seguro)
  • MTProto (utilizado por Telegram) MTProto utiliza una variedad de puertos TCP, incluidos 443, 80 y otros, según la configuración específica.
